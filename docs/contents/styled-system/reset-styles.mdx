## Customizing Reset Styles

You can customize the reset styles by simply changing the `styles.resetStyle` of the theme.

`styles.resetStyle` can define a style object or a function that returns a style object.

```tsx live=false
import { UIProvider, extendTheme, UIStyle } from "@yamada-ui/react"

const resetStyle: UIStyle = {
  th: {
    textAlign: "center",
    fontWeight: "semibold",
  },
}

const customTheme = extendTheme({ styles: { resetStyle } })()

const App = () => {
  return (
    <UIProvider theme={customTheme}>
      <YourApplication />
    </UIProvider>
  )
}
```

In the case of a function, you can get `theme` and `colorMode`.

```tsx live=false
import { UIProvider, extendTheme, UIStyle } from "@yamada-ui/react"

const resetStyle: UIStyle = ({ theme, colorMode }) => ({
  th: {
    textAlign: "center",
    fontWeight: "semibold",
  },
})

const customTheme = extendTheme({ styles: { resetStyle } })()

const App = () => {
  return (
    <UIProvider theme={customTheme}>
      <YourApplication />
    </UIProvider>
  )
}
```

:::note
If you want to learn more about themes, please check [Customize Theme](/styled-system/theming/customize-theme) and [Component Styles](/styled-system/theming/component-styles).
:::

## Adding Reset Styles

If you have chosen [Individually](/getting-started/installation/individual) or [Minimal](/getting-started/installation/minimal) during installation and installed `@yamada-ui/core`, the reset style component is not being called.

If you want to use reset styles, you need to do the following three things:

1. Add `ThemeProvider` to the root of the application.
2. Add `ResetStyle`.
3. Define the reset style and pass it to `ThemeProvider`'s `theme`.

```tsx live=false highlight={3,4,5,6,7,11,12,15}
import { ThemeProvider, ResetStyle } from "@yamada-ui/core"

const resetStyle: UIStyle = {
  "*": {
    boxSizing: "border-box",
  },
}

const App = () => {
  return (
    <ThemeProvider theme={{ styles: { resetStyle } }}>
      <ResetStyle />

      <YourApplication />
    </ThemeProvider>
  )
}
```

If you want to use the reset style of the [Default Theme](/styled-system/theming/default-theme), install `@yamada-ui/theme` and pass `defaultTheme` or `baseTheme` to `ThemeProvider`.

<PackageManagers packageNameOrCommand="@yamada-ui/theme" />

```tsx live=false highlight={7,8,11}
import { ThemeProvider, ResetStyle } from "@yamada-ui/core"
import { defaultTheme, baseTheme } from "@yamada-ui/theme"

const App = () => {
  return (
    // `defaultTheme` or `baseTheme`
    <ThemeProvider theme={baseTheme}>
      <ResetStyle />

      <YourApplication />
    </ThemeProvider>
  )
}
```

:::note
`defaultTheme` has all tokens defined, including component styles.
`baseTheme` has tokens such as global styles and colors defined, but does not have component styles defined.
:::
