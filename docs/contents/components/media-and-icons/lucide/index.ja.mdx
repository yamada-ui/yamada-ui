## インポート

<PackageManagers packageNameOrCommand="@yamada-ui/lucide" />

:::note status=warning
`@yamada-ui/lucide`は、`@yamada-ui/react`に含まれていないため、別途インストールする必要があります。
:::

## 使い方

```ts
import { Ghost } from "@yamada-ui/lucide"
```

```tsx
<Ghost />
```

### [Lucide lab](https://github.com/lucide-icons/lucide-lab)のアイコンを使用する

[Lucide lab](https://github.com/lucide-icons/lucide-lab)のアイコンを使用する場合は、`Icon`を使用します。

```ts
import { Icon as LucideIcon } from "@yamada-ui/lucide"
import { burger } from "@lucide/lab"
```

```tsx
<LucideIcon icon={burger} />
```

:::note
[Lucide lab](https://github.com/lucide-icons/lucide-lab)は、[Lucide](https://lucide.dev)に含まれていない実験的またはリリース前のアイコンのコレクションです。
:::

### 1つの汎用コンポーネント

汎用コンポーネントを1つ作成することも可能ですが、オススメしません。

```tsx live=false
const LucideIcon: FC<IconProps & { name: IconNames }> = ({ name, ...rest }) => {
  const Icon = icons[name]

  return <Icon {...rest} />
}

return <LucideIcon name="Ghost" />
```

:::note status=error
上記の例では、すべてのESモジュールをインポートしていますので、使用する際には注意が必要です。すべてのアイコンをインポートすると、アプリケーションのビルドサイズが大幅に増加し、パフォーマンスが悪化する可能性があります。これは、[webpack](https://webpack.js.org/)、[Rollup](https://rollupjs.org/)、[Vite](https://vitejs.dev/)などのバンドラを使用する際に特に重要です。
:::
