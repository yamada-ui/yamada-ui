# Branch Rules

**Format:** `<type>/<description>`

- `type` matches the [Commit Rules](./commit.md) types (`feat`, `fix`, `test`, `docs`, `refactor`, `chore`, `ci`, `build`, `perf`, `style`).
- `description` is kebab-case and briefly identifies the scope or area (component name, package, topic).

**Examples:**

```text
feat/create-component
test/color-picker-browser-mode
docs/native-popover-a11y
refactor/generate-styles
build/dependencies
```
